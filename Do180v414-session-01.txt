###  Kubernetes Architectural Concepts:-
- master nodes [Where k8s/ocp services running]
  FYI
  - 3 node cluster (w.r.t master node/control plan).
  - N worker nodes (End user applications/infra service running/deployed).
  - k8s/ocp cluster mean [master + worker nodes]
  - Container RunTime required in all nodes (master/worker) - Defult w.r.t OCP is cri-o 
  - kubelet running in all the nodes (master/worker) - it's an agent which connect with kube-apiserver and pass all information about nodes/container/API.
  - kubelet & crio both are systemd service.

Kubernetes/OCP resources:-
- POD 
- ReplicaSet  (rs - Used for replicate apps-pod accross nodes - Not with rolling update)
- Deployment  (deploy -Used for replicate apps-pod accross nodes - with rolling update)
- StatefulSet (Persistent Storage attached with Application).
- DaemonSet   (It automatically replicate apps (pods) to all the nodes (master/worker).
- Secret [ data via file/variable ; tls ; docker-registry] 
- configmap (cm)
- deploymentConfig 
- replicationController 
- cron jobs 
- jobs 
- HPA 
- PDB 
etc 

Note: 
 - DeploymentConfig & replicationController no more in use.
 - DeploymentConfig is depricated and deployment only used.
 - Deployment used replicaSet 
 - DeploymentConfig use replicationController
