-Users Details:-
  Login to VMs/Servers:-
   student/student [sudo access]
   root/redhat 
   
   
  Login to OCP cluster:- 
  admin/redhatocp  [cluster-admin access ; same as root access w.r.t Linux OS]
  developer/developer [Basic access]
  FYI- both accounts are LDAP account.
  
  kubeadmin/kubeadmin-password-file [on utility, at location - /home/lab/ocp4/auth/{kubeadmin-password, kubeconfig]
  FYI- This "kubeadmin" account created at the time of OCP cluster installation.
  Note: 
  - kubeadmin is not LDAP account as it's created at the time of OCP cluster deployment.
  - When you are trying to login to OCP cluster always use "workstation" node.
    $ oc login -u admin -p redhatocp https://api.ocp4.example.com:6443 
	$ oc whoami 
	$ oc get nodes
	$ oc get clusterversion 
	$ oc version 

-Network/Domain details:-
  Network Name                   Domain Name                    IP Range for Domain/Network
  classroom Network              example.com                    172.25.252.0/24             
  student Network                lab.example.com                172.25.250.0/24 
  Cluster Network                ocp4.example.com               192.168.50.0/24 
  
  
   $ oc login -u admin -p redhatocp https://api.ocp4.example.com:6443 
   oc - OpenShift-client (same as kubecrl ; kubectl connect to kubernetes cluster)
   FYI:-
   - oc used for connect to OCP cluster via Rest API.
   - login is API call, -u mean user, -p password, API URL=https://api.ocp4.example.com:6443 
     Note: Login method via API URL.
	 
	 As per LAB (DO180v414):-
	 API-URL =  https://api.ocp4.example.com:6443 
	         =  https://api.clusterName.baseDomain:Default-Port 
	 clusterName=ocp4 
	 baseDomain=example.com 
	 Default-Port=6443 
	----------------------------------------------------------
	 Eg:-
	 baseDomain=nokia.com 
	 
	 clusterName=apac 
	 clusterNetwork= apac.nokia.com 
	 API-URL =  https://api.apac.nokia.com:6443 
	 
	 clusterName=na 
	 API-URL =  https://api.na.nokia.com:6443 
	 clusterNetwork= apac.na.com
	 
	 clusterName=emea
     API-URL =  https://api.emea.nokia.com:6443 
     clusterNetwork= apac.emea.com
	 
    $ oc login -u admin -p redhatocp API-URL 
    $ oc login -u admin -p redhatocp https://api.apac.nokia.com:6443
	$ oc login -u admin -p redhatocp https://api.na.nokia.com:6443
	$ oc login -u admin -p redhatocp https://api.apac.emea.com:6443

  classroom.example.com   - It allow to connect to internet for access public/private registry/repo

  - etc 
  
  utility.lab.example.com - Main node which provide very imp services [DNS/HAproxy/DHCP/PXE/Apache/NFS-Server-for-StorageCLass etc]
